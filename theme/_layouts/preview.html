--- layout: default --- {% comment %} Live preview shell. Rendered with the real site chrome (default layout) so typography, header, footer, and CSS are identical to production by construction — changing a layout or stylesheet automatically flows into the preview without a separate code path. The shell hosts three hidden layout fragments — post, page, project — mirroring the field structure of each Jekyll layout. The client script picks one based on ?collection=, exposes named slots via [data-preview-slot="…"], and streams draft content in from the CMS via window.postMessage({ type: "cms-preview-update", fields: {…} }). Security: all messages are origin-checked against the preview page's own origin, so only same-origin callers (the CMS admin) can update it. {% endcomment %}
